Attack Explanation

1 Victim visits this attacker-controlled page (any website on the internet)
2 JavaScript makes a cross-origin POST to https://stake.com/_api/graphql
3 No authentication headers sent — no cookies, no tokens
4 Server returns victim's real IP address with Access-Control-Allow-Origin: *
5 Attacker's page reads the response and exfiltrates the IP

Why This Matters

  • IP addresses are PII under GDPR Article 4(1) / Recital 30
  • Deanonymizes users behind VPNs/Tor (Stake.com sees their real connection IP)
  • Enables geolocation tracking without consent
  • No user interaction required — silent, invisible to the victim
  • Any website can do this — Access-Control-Allow-Origin: * allows all origins
